# Git 工作流参考 - OpenClaw 本地开发

## 分支策略

- `main` - 保持与 origin/main 同步（纯净镜像）
- `my-local-improvements` - 你的长期开发分支
- `feature/*` - 单独特性分支（用于提PR）

## 常用命令

### 日常开发

```bash
# 切换到开发分支
git checkout my-local-improvements

# 提交修改
git add <files>
git commit -m "feat: description"

# 查看状态
git status
git log --oneline --graph -10
```

### 同步远程更新（每天/每周）

```bash
# 一键同步脚本
git checkout main && \
git pull origin main && \
git checkout my-local-improvements && \
git merge main

# 如果有冲突
git status                    # 查看冲突文件
# ... 手动解决冲突 ...
git add <resolved-files>
git commit                    # 完成合并
```

### 使用Rebase（可选，更干净）

```bash
# 在开发分支上
git checkout my-local-improvements
git fetch origin
git rebase origin/main

# 如果有冲突
# ... 解决冲突 ...
git add <files>
git rebase --continue

# 如果想放弃rebase
git rebase --abort
```

### 分支管理

```bash
# 查看所有分支
git branch -a

# 创建新特性分支
git checkout -b feature/new-feature

# 删除已合并的分支
git branch -d branch-name

# 强制删除分支
git branch -D branch-name

# 查看分支差异
git diff main my-local-improvements
```

### 暂存工作（临时切换分支）

```bash
# 保存当前工作
git stash save "WIP: description"

# 切换到其他分支工作
git checkout main

# 恢复工作
git checkout my-local-improvements
git stash pop
```

### 撤销操作

```bash
# 撤销未提交的修改
git restore <file>

# 撤销已暂存的文件
git restore --staged <file>

# 撤销最后一次提交（保留修改）
git reset --soft HEAD~1

# 撤销最后一次提交（丢弃修改）
git reset --hard HEAD~1
```

### 查看变更

```bash
# 查看未提交的修改
git diff

# 查看已暂存的修改
git diff --staged

# 查看两个分支的差异
git diff main..my-local-improvements

# 查看文件修改历史
git log -p <file>
```

## 冲突解决流程

```bash
# 1. 尝试合并
git merge main

# 2. 如果有冲突
git status                           # 查看冲突文件

# 3. 手动编辑冲突文件，移除标记：
#    <<<<<<< HEAD
#    你的代码
#    =======
#    远程代码
#    >>>>>>> main

# 4. 标记为已解决
git add <resolved-files>

# 5. 完成合并
git commit -m "merge: sync with upstream"
```

## 最佳实践

1. **保持main分支纯净** - 永远不要直接在main上开发
2. **频繁提交** - 小步提交，方便回滚和查找问题
3. **有意义的提交信息** - 使用 `feat:`, `fix:`, `refactor:` 等前缀
4. **定期同步** - 每天或每周从远程拉取更新
5. **解决冲突时小心** - 理解你的代码和远程代码的意图
6. **备份重要修改** - 创建分支前先 `git stash` 或提交

## 紧急情况

### 误删除重要代码

```bash
# 查看最近的提交
git reflog

# 恢复到某个提交
git reset --hard <commit-hash>
```

### 合并出错想重来

```bash
# 放弃合并
git merge --abort

# 或回退到合并前
git reset --hard HEAD~1
```

### 想临时测试远程最新代码

```bash
# 切换到main（远程镜像）
git checkout main
git pull origin main

# 测试完成后切回开发分支
git checkout my-local-improvements
```
